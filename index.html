1. MEMAHAMI WIRELESS CLIENT ISOLATION

Ilustrasi topologi jaringan untuk isolasi client wireless di MikroTik

Pengertian Client Isolation

Dari sumber MikroTik Documentation (2024), client isolation adalah fitur keamanan di jaringan wireless yang mencegah perangkat client yang terhubung ke Access Point (AP) yang sama untuk berkomunikasi satu sama lain. Fitur ini juga dikenal sebagai "client-to-client blocking" atau "AP isolation". Ketika dihidupkan, masing-masing client wireless hanya dapat berkomunikasi dengan router (access point) dan jaringan eksternal, tetapi tidak dapat melihat atau berkomunikasi langsung dengan client wireless lainnya yang terhubung ke AP yang sama.

Dari sumber Tangent Soft MikroTik Wiki (2024), isolasi client berfungsi untuk menjaga keamanan dengan memastikan bahwa perangkat tamu atau IoT tidak dapat saling memata-matai atau saling mengirim perintah. Jika ada dua host yang perlu berkomunikasi satu sama lain, mereka seharusnya tidak berada di jaringan "tamu" yang terisolasi.

Manfaat Client Isolation

  • Meningkatkan Keamanan: Mencegah serangan man-in-the-middle antar perangkat dan aktivitas sniffing antar client.
  • Privasi Pengguna: Melindungi data pengguna dari pengintaian oleh pengguna lain dalam jaringan yang sama.
  • Ideal untuk Jaringan Publik: Sangat cocok untuk hotspot publik, kafe, hotel, atau area dengan banyak pengguna yang tidak saling mengenal.
  • Mencegah Penyebaran Malware: Mengurangi risiko penyebaran malware antar perangkat dalam jaringan.
  • Kontrol Sumber Daya: Memastikan pengguna hanya mengakses sumber daya yang diizinkan melalui router.

Cara Kerja Client Isolation di MikroTik

Dari sumber Forum MikroTik (2023), terdapat beberapa pendekatan untuk mengimplementasikan isolasi client wireless di perangkat MikroTik:

# 1. Client Isolation pada tingkat interface wireless
Mengaktifkan properti datapath.client-isolation di konfigurasi interface wireless
# 2. Bridge Filtering
Menggunakan filter pada bridge untuk mencegah forwarding antar port tertentu
# 3. VLAN Separation
Menggunakan VLAN berbeda untuk setiap client atau kelompok client
# 4. Firewall Rules
Membuat aturan firewall untuk memblokir komunikasi antar subnet client

Dari sumber BlockDev.io (2022), pendekatan konvensional untuk jaringan tamu sering dibuat pada level wireless AP dan menggunakan client isolation untuk memastikan client wireless tidak dapat saling berkomunikasi. Namun untuk isolasi yang lebih komprehensif, isolasi pada level jaringan (subnet terpisah) dapat memberikan kontrol yang lebih baik.

Dari 2 pendapat tersebut dapat disimpulkan bahwa client isolation adalah fitur keamanan penting di jaringan wireless yang mencegah komunikasi langsung antar client, dengan beberapa metode implementasi di MikroTik mulai dari level wireless interface hingga level jaringan yang lebih luas. Pemilihan metode bergantung pada kebutuhan keamanan dan kompleksitas jaringan.

2. MENGONFIGURASI CLIENT ISOLATION PADA LEVEL INTERFACE

Tampilan konfigurasi client isolation di MikroTik RouterOS (Winbox)
Tampilan konfigurasi client isolation di MikroTik RouterOS (Winbox)

Metode Dasar Client Isolation

Dari sumber Tangent Soft MikroTik Wiki (2024), konfigurasi dasar untuk client isolation di MikroTik dapat dilakukan dengan menggunakan properti datapath.client-isolation pada konfigurasi wireless. Fitur ini akan mencegah client wireless yang terhubung ke Access Point yang sama untuk dapat berkomunikasi satu sama lain.

Dari sumber Forum MikroTik (2021), pengaturan default-forwarding (pada versi RouterOS lama) atau datapath.client-isolation (pada RouterOS v7+) hanya memblokir forwarding antar client ketika kedua client dilayani oleh AP yang sama. Jika ingin memblokir konektivitas antar client dari AP yang berbeda, harus menggunakan bridge filtering atau subnet yang berbeda melalui VLAN.

Konfigurasi Client Isolation via CLI

Berikut adalah langkah-langkah konfigurasi client isolation pada interface wireless menggunakan Command Line Interface (CLI) MikroTik:

# Untuk RouterOS versi 7 ke atas
/interface wifi configuration add name="guestcfg" ssid="Guest-WiFi" \
datapath.client-isolation=yes \
security.authentication-types=wpa2-psk \
security.passphrase="GuestPassword123"
/interface wifi add name="wifi-guest" configuration="guestcfg" \
master-interface=wlan1

Untuk mengecek apakah client isolation sudah aktif pada interface wireless:

/interface wifi configuration print where name="guestcfg"

Perhatikan bahwa parameter yang digunakan berbeda antara RouterOS versi 6 dan versi 7+. Pada RouterOS v6, properti untuk isolasi client adalah default-forwarding=no, sedangkan pada RouterOS v7, properti yang digunakan adalah datapath.client-isolation=yes.

Pengujian Client Isolation

Untuk memverifikasi bahwa client isolation berfungsi dengan benar, Anda dapat melakukan pengujian berikut:

# 1. Hubungkan dua atau lebih perangkat ke SSID tamu
# 2. Cek perangkat yang terhubung di MikroTik
/interface wifi registration-table print
# Hasil contoh:
# INTERFACE MAC-ADDRESS IP-ADDRESS UPTIME
# wifi-guest AA:BB:CC:11:22:33 192.168.99.2 10m32s
# wifi-guest DD:EE:FF:44:55:66 192.168.99.3 5m10s
# 3. Dari satu client, coba ping client lain
# Jika client isolation berfungsi, ping akan gagal

Dari 2 pendapat tersebut dapat disimpulkan bahwa mengonfigurasi client isolation pada level interface wireless di MikroTik relatif sederhana dengan menggunakan properti datapath.client-isolation (RouterOS v7+) atau default-forwarding (RouterOS v6). Namun, perlu diingat bahwa metode ini hanya mencegah komunikasi antar client yang terhubung ke AP yang sama, sehingga diperlukan pendekatan tambahan jika ingin isolasi yang lebih komprehensif.

3. ISOLASI CLIENT MENGGUNAKAN BRIDGE FILTERING

Konsep Bridge Filtering

Dari sumber Forum MikroTik (2023), bridge filtering dapat digunakan untuk membatasi komunikasi antar perangkat pada level layer 2 (MAC). Metode ini lebih fleksibel dan dapat digunakan untuk memblokir konektivitas antara client yang terhubung ke AP yang berbeda tetapi masih berada dalam bridge yang sama.

Dari sumber Tangent Soft MikroTik Wiki (2024), pendekatan bridge filtering menggunakan filter pada level bridge daripada filter IP, dan mencegah forwarding antar tamu dan juga ke jaringan lain. Ini dapat menjadi solusi alternatif yang efektif untuk isolasi client tanpa perlu menggunakan VLAN.

Konfigurasi Bridge Filtering via CLI

Langkah-langkah untuk mengonfigurasi isolasi client menggunakan bridge filtering:

# 1. Buat bridge untuk jaringan tamu
/interface bridge add name=guest-bridge
# 2. Tambahkan interface wireless ke bridge
/interface bridge port add bridge=guest-bridge interface=wifi-guest
# 3. Aktifkan bridge filter
/interface bridge set guest-bridge protocol-mode=none
# 4. Buat aturan bridge filter untuk memblokir komunikasi antar client
/interface bridge filter add chain=forward action=drop in-bridge=guest-bridge \
out-bridge=guest-bridge mac-protocol=ip
# 5. Izinkan akses ke router (untuk DHCP, DNS, dll)
/interface bridge filter add chain=forward action=accept in-bridge=guest-bridge \
dst-mac-address=XX:XX:XX:XX:XX:XX mac-protocol=ip
# Ganti XX:XX:XX:XX:XX:XX dengan MAC address interface bridge router

Untuk memeriksa aturan bridge filter yang telah dikonfigurasi:

/interface bridge filter print
# Hasil contoh:
# Flags: X - disabled, I - invalid, D - dynamic
# 0 chain=forward action=drop in-bridge=guest-bridge out-bridge=guest-bridge mac-protocol=ip
# 1 chain=forward action=accept in-bridge=guest-bridge dst-mac-address=XX:XX:XX:XX:XX:XX mac-protocol=ip

Pastikan untuk selalu mengizinkan akses ke router (gateway) agar client tetap dapat mengakses internet dan layanan jaringan seperti DHCP dan DNS. Tanpa akses ini, client akan terisolasi secara total dan tidak dapat melakukan koneksi apapun.

Keuntungan Bridge Filtering

  • Isolasi Komprehensif: Dapat memblokir komunikasi antar semua client pada bridge yang sama, terlepas dari AP mana mereka terhubung.
  • Fleksibilitas: Memungkinkan pengecualian untuk perangkat tertentu berdasarkan MAC address.
  • Performa: Filtering pada layer 2 umumnya lebih efisien dibandingkan filtering pada layer 3 (IP).
  • Tidak memerlukan VLAN: Solusi yang lebih sederhana untuk jaringan kecil tanpa perlu menggunakan VLAN.

Dari 2 pendapat tersebut dapat disimpulkan bahwa bridge filtering menawarkan pendekatan yang lebih komprehensif untuk isolasi client dibandingkan dengan isolasi pada level interface wireless saja. Metode ini dapat memblokir komunikasi antar client yang terhubung ke AP yang berbeda selama mereka berada dalam bridge yang sama, sehingga memberikan lapisan keamanan tambahan untuk jaringan wireless.

4. ISOLASI CLIENT DENGAN VLAN DAN SUBNET TERPISAH

Konsep Isolasi dengan VLAN

Dari sumber Forum MikroTik (2024), penggunaan VLAN adalah pendekatan yang sangat efektif untuk isolasi client wireless yang memungkinkan pemisahan trafik pada level jaringan. Dengan menempatkan setiap SSID atau bahkan setiap client pada VLAN yang berbeda, administrator dapat memastikan isolasi yang lebih lengkap dan menerapkan kebijakan firewall yang spesifik untuk setiap kelompok.

Dari sumber BlockDev.io (2022), isolasi dengan subnet terpisah memungkinkan client pada jaringan tamu untuk saling melihat tetapi tidak dapat melihat jaringan utama. Pendekatan ini memberikan level isolasi yang lebih tinggi dibandingkan dengan client isolation sederhana pada level AP.

Konfigurasi VLAN dan Subnet untuk Jaringan Tamu

Berikut adalah langkah-langkah untuk mengonfigurasi isolasi client menggunakan VLAN dan subnet terpisah:

# 1. Buat VLAN interface untuk jaringan tamu
/interface vlan add name=vlan-guest vlan-id=100 interface=ether1
# 2. Buat bridge untuk jaringan tamu
/interface bridge add name=bridge-guest
# 3. Tambahkan VLAN interface dan interface wireless ke bridge
/interface bridge port add bridge=bridge-guest interface=vlan-guest
/interface bridge port add bridge=bridge-guest interface=wifi-guest
# 4. Konfigurasi IP address untuk subnet tamu
/ip address add address=192.168.99.1/24 interface=bridge-guest
# 5. Konfigurasi DHCP server untuk jaringan tamu
/ip pool add name=guest-pool ranges=192.168.99.100-192.168.99.200
/ip dhcp-server add name=guest-dhcp interface=bridge-guest address-pool=guest-pool
# 6. Konfigurasi network dan gateway untuk DHCP
/ip dhcp-server network add address=192.168.99.0/24 gateway=192.168.99.1 dns-server=8.8.8.8

Konfigurasi Firewall untuk Isolasi Subnet

Setelah membuat VLAN dan subnet terpisah, tambahkan aturan firewall untuk memastikan isolasi:

# 1. Buat address-list untuk jaringan tamu
/ip firewall address-list add list=guest-network address=192.168.99.0/24
# 2. Izinkan koneksi yang sudah established/related
/ip firewall filter add chain=forward action=accept connection-state=established,related
# 3. Blokir akses dari jaringan tamu ke jaringan lokal
/ip firewall filter add chain=forward action=drop src-address-list=guest-network \
dst-address=192.168.88.0/24 comment="Block guest to LAN"
# 4. Blokir komunikasi antar client di jaringan tamu
/ip firewall filter add chain=forward action=drop src-address-list=guest-network \
dst-address-list=guest-network comment="Block guest to guest"
# 5. Izinkan akses dari jaringan tamu ke internet
/ip firewall filter add chain=forward action=accept src-address-list=guest-network \
connection-state=new comment="Allow guest to internet"
# 6. Konfigurasi NAT untuk jaringan tamu
/ip firewall nat add chain=srcnat action=masquerade src-address-list=guest-network

Pendekatan VLAN memungkinkan isolasi yang lebih granular. Anda bahkan dapat membuat VLAN terpisah untuk kelompok perangkat yang berbeda, seperti perangkat IoT, tamu bisnis, dan tamu umum, masing-masing dengan kebijakan keamanan yang berbeda.

Keuntungan Penggunaan VLAN dan Subnet Terpisah

  • Isolasi Lengkap: Pemisahan pada level jaringan memberikan isolasi yang lebih komprehensif.
  • Kontrol Akses Granular: Memungkinkan kebijakan firewall yang spesifik untuk setiap kelompok client.
  • Manajemen QoS: Memudahkan penerapan kebijakan Quality of Service per VLAN.
  • Skalabilitas: Lebih mudah ditingkatkan untuk jaringan yang lebih besar.
  • Keamanan Tingkat Tinggi: Isolasi pada multiple layer (L2 dan L3) memberikan keamanan yang lebih baik.

Dari 2 pendapat tersebut dapat disimpulkan bahwa penggunaan VLAN dan subnet terpisah menawarkan pendekatan isolasi client yang paling komprehensif dan fleksibel. Meskipun memerlukan konfigurasi yang lebih kompleks, metode ini memberikan kontrol akses yang lebih granular dan keamanan yang lebih tinggi, terutama untuk jaringan enterprise atau yang memerlukan kebijakan keamanan yang ketat.

5. MENGATUR ACCESS CONTROL DENGAN ACCESS LIST

Konsep Access List di MikroTik

Dari sumber MikroTik Documentation (2024), access list pada RouterOS menyediakan berbagai cara untuk melakukan filtering dan mengelola koneksi wireless. RouterOS akan memeriksa setiap koneksi baru untuk melihat apakah parameternya cocok dengan parameter yang ditentukan dalam aturan access list. Aturan diperiksa sesuai urutan yang muncul dalam daftar, dan hanya tindakan manajemen yang ditentukan dalam aturan pertama yang cocok yang diterapkan pada setiap koneksi.

Dari sumber MikroTik Wiki (2023), access list memungkinkan administrator untuk mengendalikan perangkat mana yang dapat terhubung ke jaringan wireless berdasarkan MAC address, waktu koneksi, dan tingkat sinyal. Koneksi yang telah diterima oleh aturan access list akan diperiksa secara berkala untuk memastikan mereka tetap berada dalam rentang waktu dan sinyal yang diizinkan.

Konfigurasi Access List via CLI

Langkah-langkah untuk mengonfigurasi access control menggunakan access list di MikroTik:

# 1. Tambahkan aturan untuk mengizinkan perangkat berdasarkan MAC address
/interface wifi access-list add interface=wifi-guest mac-address=AA:BB:CC:11:22:33 \
comment="Device John's laptop" action=accept
# 2. Tambahkan aturan untuk membatasi waktu akses
/interface wifi access-list add interface=wifi-guest mac-address=DD:EE:FF:44:55:66 \
comment="Limited time access" action=accept \
start-time=8:00:00 end-time=17:00:00
# 3. Tambahkan aturan untuk menolak perangkat tertentu
/interface wifi access-list add interface=wifi-guest mac-address=11:22:33:44:55:66 \
comment="Blocked device" action=reject
# 4. Tambahkan aturan default untuk menolak semua koneksi lain
/interface wifi access-list add interface=wifi-guest \
comment="Default reject rule" action=reject

Untuk memeriksa access list yang telah dikonfigurasi:

/interface wifi access-list print
# Hasil contoh:
# Flags: X - disabled
# 0 interface=wifi-guest mac-address=AA:BB:CC:11:22:33 action=accept comment="Device John's laptop"
# 1 interface=wifi-guest mac-address=DD:EE:FF:44:55:66 action=accept start-time=8:00:00 end-time=17:00:00 comment="Limited time access"
# 2 interface=wifi-guest mac-address=11:22:33:44:55:66 action=reject comment="Blocked device"
# 3 interface=wifi-guest action=reject comment="Default reject rule"

Berhati-hatilah saat membuat aturan access list yang menolak client. Seperti disebutkan dalam dokumentasi MikroTik, setelah berulang kali ditolak oleh AP, perangkat client mungkin mulai menghindarinya. Pastikan untuk menguji aturan access list dengan hati-hati untuk menghindari masalah konektivitas.

Access Control dengan RADIUS

Untuk deployment yang lebih besar, MikroTik mendukung autentikasi RADIUS untuk client wireless:

# 1. Konfigurasi server RADIUS
/radius add service=wireless address=10.0.0.5 secret="radiusSecret"
# 2. Aktifkan autentikasi RADIUS MAC untuk interface wireless
/interface wifi set wifi-guest radius-mac-authentication=yes \
radius-mac-format=XX:XX:XX:XX:XX:XX radius-mac-mode=as-username

Kombinasi Access List dengan Isolasi Client

Menggabungkan access list dengan teknik isolasi client memberikan kontrol penuh atas siapa yang dapat terhubung dan bagaimana mereka dapat berkomunikasi:

# 1. Konfigurasi isolasi client
/interface wifi configuration set [find where name="guestcfg"] \
datapath.client-isolation=yes
# 2. Konfigurasi access list untuk kontrol akses
/interface wifi access-list add interface=wifi-guest mac-address=AA:BB:CC:11:22:33 \
action=accept
/interface wifi access-list add interface=wifi-guest action=reject

Dari 2 pendapat tersebut dapat disimpulkan bahwa access list di MikroTik adalah alat yang kuat untuk mengelola akses ke jaringan wireless dengan memberikan kontrol granular berdasarkan MAC address, waktu, dan parameter lainnya. Dengan menggabungkan access list dengan teknik isolasi client, administrator dapat menciptakan jaringan wireless yang aman dengan kontrol penuh atas siapa yang dapat terhubung dan bagaimana mereka dapat berkomunikasi di dalam jaringan.
